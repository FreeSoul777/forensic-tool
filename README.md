# Forensic Tool

`Создано в учебных целях`

Forensic Tool - утилита для поиска артефактов удаленных пользователей в Linux-системах. 
Инструмент анализирует систему и собирает информацию о процессах, сетевых соединениях, 
cron-задачах, файлах и логах, которые могли остаться после удаления пользователей.

## Возможности

- `Поиск удаленных пользователей` - обнаружение процессов, файлов и задач, принадлежащих пользователям, которых нет в /etc/passwd
- `Анализ сети` - поиск открытых портов и Unix-сокетов от удаленных пользователей
- `Сканирование процессов` - обнаружение запущенных процессов от удаленных пользователей
- `Поиск задач` - анализ cron, systemd таймеров, at и anacron задач
- `Сканирование файловой системы` - поиск файлов, оставшихся после удаления пользователей
- `Анализ логов` - поиск записей в логах, связанных с удаленными пользователями
- `Генерация отчетов` - создание отчетов в форматах JSON, HTML, PDF
- `Извлечение артефактов` - копирование найденных файлов с вычислением хэш-сумм (MD5, SHA256)

## Требования

- Linux (тестировалось на Ubuntu 20.04/22.04, Debian)
- Python 3.8+
- Bash
- Root-права (для доступа к системным директориям)

## Установка

### Клонирование репозитория

```bash
git clone https://github.com/yourusername/forensic-tool.git
cd forensic-tool
```

### Установка зависимостей

```bash
pip install -r requirements.txt
```

Для поддержки PDF отчетов (опционально):
```bash
pip install fpdf2
```

## Запуск

Утилита требует root-прав:

```bash
sudo python -m forensic
```

### Параметры командной строки

- `--log-level`, `-l` - уровень логирования (DEBUG, INFO, WARNING, ERROR)
- `--session-dir`, `-s` - директория для сохранения сессий
- `--version` - показать версию

Пример:
```bash
sudo python -m forensic --log-level DEBUG
```

## Меню программы

После запуска доступны следующие действия:

1. `Сканировать систему` - поиск удаленных пользователей и сбор артефактов
2. `Конвертировать отчет` - конвертация JSON отчета в HTML или PDF
3. `Распарсить отчет` - извлечение файловых артефактов с хэш-суммами
4. `Выход`

### 1. Сканирование системы

При выборе этого пункта утилита выполняет полное сканирование системы:
- Сбор информации о системе (хост, ОС, ядро)
- Поиск удаленных пользователей по 6 направлениям
- Сохранение отчета в формате JSON в директории сессии

Результат сохраняется в: `/tmp/forensic_sessions/YYYYMMDD_HHMMSS_username/report.json`

### 2. Конвертация отчета

Позволяет конвертировать JSON отчет в HTML или PDF форматы:
- Укажите путь до JSON файла
- Выберите целевой формат (html или pdf)

### 3. Распарсивание отчета

Извлекает файловые артефакты из отчета:
- Для каждого удаленного пользователя создается отдельная папка `uid_*`
- В папке сохраняется `personal_report.json` (копия данных пользователя)
- Все найденные файлы копируются в `artifacts.zip` внутри папки пользователя
- Для каждого файла вычисляются MD5 и SHA256 хэши

## Структура проекта

```
forensic-tool/
├── forensic/
│   ├── core/              # Ядро (логгер, настройки)
│   ├── scanners/          # Сканеры (bash-скрипт и обертка)
│   ├── report/            # Генерация отчетов (JSON, HTML, PDF)
│   ├── system/            # Системная информация
│   └── utils/             # Утилиты (хэширование, извлечение)
├── config/                 # Конфигурационные файлы
├── tests/                  # Тесты
├── requirements.txt        # Зависимости Python
├── Makefile                # Команды для Docker
├── docker/                 # Dockerfile
└── README.md               # Этот файл
```

## Конфигурация

Файл конфигурации `config/forensic.conf`:

```ini
[forensic]
# Уровень логирования: DEBUG, INFO, WARNING, ERROR
log_level = INFO

# Директория для сессий
session_dir = /tmp/forensic_sessions

# Максимальный размер лог-файла в байтах (10 MB)
max_log_size = 10485760

# Сколько хранить backup логов
log_backup_count = 5

# Сколько дней хранить сессии (0 - вечно)
session_retention_days = 5
```
